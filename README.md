## ğŸ”„ CI/CD Workflow
The complete CI/CD flow implemented in this project is as follows:

1. Developer pushes source code to GitHub.
2. Jenkins automatically pulls the latest code from GitHub.
3. Maven builds the application and generates a WAR file.
4. Docker image is created using the WAR file.
5. Ansible playbooks automate Docker image build and push to Docker Hub.
6. Kubernetes pulls the Docker image from Docker Hub.
7. Application is deployed as pods using Kubernetes Deployment.
8. Kubernetes Service (LoadBalancer) exposes the application to users.
